Info tables display static data, and editable tables, are editable (like a
spreadsheet).

The header of both these types of tables is the same. You can use the sort
icons to sort columns in descending or ascending order. By default these
columns are unsorted.

<GuidelineImage
  imageURL={`${CDN}/images/new-guidelines/infoTable/infoTable-guidelines-1@3x.png`}
  className={css`
    width: auto;
    height: 41px;
  `}
/>

A menu trigger can be added to each column header, for example to allow the
user to delete a column.

<GuidelineImage
  imageURL={`${CDN}/images/new-guidelines/infoTable/infoTable-guidelines-2@3x.png`}
  className={css`
    width: auto;
    height: 87px;
  `}
/>

When your info table spans more than 10 rows you should alternate the
background color of each row, so that your table is more scannable.

<GuidelineImage
  imageURL={`${CDN}/images/new-guidelines/infoTable/infoTable-guidelines-3@3x.png`}
  className={css`
    height: 520px;
    ${mq({
      width: ['auto', 'auto', 'auto', '946px'],
    })}
  `}
/>

Do not use the alternating background pattern when there are fewer than 10
line items in your table. Otherwise it may look like one of the rows is
selected or disabled.

<GuidelineImage
  imageURL={`${CDN}/images/new-guidelines/infoTable/infoTable-guidelines-4@3x.png`}
  className={css`
    height: 179px;
    ${mq({
      width: ['auto', 'auto', 'auto', '1053px'],
    })}
  `}
/>

### Content alignment

Text should be left aligned. Numbers expressing quantity should be right
aligned. Dates can be either left or right aligned.{' '}

[Learn more about content alignment.](https://practicaltypography.com/grids-of-numbers.html)

### Disabled Rows

Disabled rows have a darker background and lighter copy and show the disabled cursor
on hover. Do not use these cells alongside alternate colored rows as this will make
it difficult to distinguish which row is disabled and which is not.

<GuidelineImage
  imageURL={`${CDN}/images/new-guidelines/infoTable/infoTable-guidelines-6@3x.png`}
  className={css`
    height: 265px;
    ${mq({
      width: ['auto', 'auto', 'auto', '1098px'],
    })}
  `}
/>

### Nesting

Avoid nesting tables within other tables or within other elements. These are both
challenging to read and build.
